# UI 设计规范 - 原研哉美学

Zerone 界面设计遵循原研哉（Kenya Hara）的设计哲学：虚空、留白、减法、触知。本文档为阶段一原型的落地规范。

---

## 一、核心理念映射

| 原研哉理念 | Zerone 落地 |
|------------|-------------|
| 虚空 / 留白 | 大面积留白，信息密度低，聚焦当前任务 |
| 白 | 主背景为浅白/米白，避免纯黑背景；强调「空」的包容性 |
| 减法 | 去掉非必要元素，仅保留核心操作与内容 |
| 触知 | 按钮、输入框有清晰的 hover/focus 反馈，有「可触碰」的暗示 |
| 情感传达 | 「从 0 到 1」通过留白与简约传达创造感，而非科技感堆砌 |

---

## 二、色彩

| 用途 | 色值 | 说明 |
|------|------|------|
| 主背景 | `#fafafa` / `#f5f5f0` | 米白或浅灰，避免纯白刺眼 |
| 次级背景 | `#f0f0f0` / `#e8e8e8` | 卡片、输入框底色 |
| 主文字 | `#333` / `#2c2c2c` | 深灰，无纯黑 |
| 次要文字 | `#666` / `#8b8b8b` | 说明、标签 |
| 边框 | `#e0e0e0` | 细线，颜色淡 |
| 强调色 | `#4a7ab8` | 链接、主按钮，极克制 |
| 强调悬停 | `#5a8ac8` | 略亮于强调色 |

**原则**：无渐变、无高饱和色、无大面积彩色。

---

## 三、字体

| 用途 | 规范 |
|------|------|
| 主字体 | 无衬线（如 -apple-system, BlinkMacSystemFont, "PingFang SC", "Helvetica Neue"） |
| 字重 | regular (400)、medium (500)，避免过多层级 |
| 字号层级 | 3–4 级：标题 24–28px、副标题 16–18px、正文 14–15px、辅助 12px |
| 行高 | 正文 1.5–1.6，标题 1.2–1.3 |
| 代码 | 等宽字体（JetBrains Mono、Menlo、monospace） |

---

## 四、间距

| 场景 | 规范 |
|------|------|
| 页面边距 | 24–32px |
| 模块间距 | 32–48px |
| 卡片内边距 | 16–24px |
| 元素间距 | 8–12px |

**原则**：模块间留白大于内容区，整体偏「空」。

---

## 五、组件风格

- **卡片/容器**：细线边框 `1px solid #e0e0e0`，无阴影或极浅投影（可选 `0 1px 2px rgba(0,0,0,0.04)`）
- **按钮**：扁平，主按钮用强调色填充，次要按钮用边框
- **输入框**：细边框，focus 时边框色加深
- **无**：渐变、阴影、复杂圆角、图标滥用

---

## 六、页面结构

### 工作台首页
- 居中布局，上下留白大
- 标题 Zerone + 副标题
- 2 个模块卡片，横向排列，间距充足
- 无装饰性图形

### AI 对话页
- 左侧窄边栏（会话列表，约 200px）
- 主区：消息列表 + 底部输入框
- 消息气泡：极简，区分用户/AI 背景色
- 输入框：简洁，无过多占位

### 代码生成页
- 上：需求输入区（单行或多行输入）
- 下：代码展示区（等宽字体、语法高亮）
- 复制按钮：低调，靠近代码区

---

## 七、交互反馈

- **hover**：边框或背景色轻微变化
- **focus**：可见焦点环（outline 或 border）
- **loading**：打字光标或轻微脉动，避免旋转 loading 图标
- **错误**：文字提示 + 重试入口，不打断整体布局
