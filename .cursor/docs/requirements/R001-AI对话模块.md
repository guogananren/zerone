# R001 - AI 对话模块

## 需求概述

基于阿里云百炼大模型，实现多轮 AI 机器人对话，支持流式输出与会话管理。

## 功能要点

| 功能 | 说明 |
|------|------|
| 聊天界面 | 用户消息 / AI 回复，气泡式布局 |
| 流式打字 | 回复逐字展示，生成时显示光标 |
| 会话管理 | 新建会话、切换历史会话、会话列表左侧展示 |
| BFF 代理 | 百炼 API 由 BFF 转发，API Key 不暴露于前端 |

## 验收标准

- [ ] 输入消息后可获得 AI 回复
- [ ] 回复以流式逐字展示
- [ ] 支持新建会话、切换历史会话
- [ ] 多轮对话保持上下文
- [ ] 错误时有友好提示与重试

## 技术实现

### 架构

```
浏览器 (ChatBot.vue)
    ↓ fetch('/api/chat')
Vite Dev Server (反向代理 /api → localhost:3001)
    ↓ 转发
BFF (Express, server/index.js)
    ↓ OpenAI SDK
阿里云百炼 API (dashscope.aliyuncs.com)
```

### 模块与路径

| 层级 | 路径 | 说明 |
|------|------|------|
| 页面 | `@/pages/chat/index.vue` | 聊天页容器 |
| 组件 | `@/pages/chat/components/ChatBot.vue` | 聊天 UI、输入、侧栏 |
| 状态 | `@/stores/chat.ts` | Pinia Store，会话与消息 |
| 接口 | `@@/apis/chat.ts` | `fetchChatStream` 流式请求 |
| BFF | `server/index.js` | Express，使用 OpenAI SDK 调用百炼 |
| 环境 | `.env` | `DASHSCOPE_API_KEY` |

### 关键接口

**BFF `/api/chat`**

- 方法：POST
- 请求体：`{ messages: { role, content }[] }`
- 响应：SSE 流，`data: {"content":"..."}\n\n`，结尾 `data: [DONE]\n\n`

**百炼调用**

- SDK: `openai`（OpenAI 兼容）
- baseURL: `https://dashscope.aliyuncs.com/compatible-mode/v1`
- 模型: `qwen-plus`
- 流式: `stream: true`

## 实现逻辑

### 一、客户端：ChatBot.vue

**1. 发送触发**

用户点击发送或按 Enter 时进入 `handleSend`：

1. `chatStore.ensureCurrentSession()`：确保当前会话存在
2. `chatStore.addMessage(session.id, userMessage)`：写入用户消息
3. `chatStore.addMessage(session.id, botPlaceholder)`：新增空 bot 占位（供流式追加）
4. `buildApiMessages()`：将 Store 消息转为 `{ role, content }[]`，过滤空 bot
5. `fetchChatStream(apiMessages, onChunk)`：发起请求并处理流

**2. 消息格式转换**

`buildApiMessages` 将 `currentMessages` 中 `sender: 'user'|'bot'` 映射为 `role: 'user'|'assistant'`，仅包含有内容的消息（空 bot 占位被过滤）。

### 二、API 层：fetchChatStream

**1. 发起请求**

```ts
fetch('/api/chat', {
  method: 'POST',
  headers: { 'Content-Type': 'application/json' },
  body: JSON.stringify({ messages }),
})
```

请求发往当前页面域名（开发时为 Vite Dev Server，如 `http://localhost:5173`），路径 `/api/chat` 被 Vite 代理转发。

**2. 错误处理**

`resp.ok === false` 时解析 `resp.json()` 的 `error` 字段并抛出，由 ChatBot 的 `catch` 写入 `errorMsg` 并调用 `updateLastBotMessage` 展示错误文案。

**3. SSE 流解析**

使用 `resp.body.getReader()` 读取 ReadableStream，按行拆分，识别 `data: ` 前缀，`data: [DONE]` 表示结束，其余解析 JSON 取 `content` 后回调 `onChunk(content)`。

### 三、Vite 反向代理

`vite.config.ts` 中配置：

```ts
proxy: {
  '/api': { target: 'http://localhost:3001', changeOrigin: true },
}
```

- 所有以 `/api` 开头的请求转发到 `http://localhost:3001`
- BFF 的响应（含 SSE）原样返回给浏览器

### 四、BFF：server/index.js

**1. 环境加载**

通过 `dotenv` 从项目根 `.env` 加载 `DASHSCOPE_API_KEY`。

**2. 请求校验**

- 无 API Key 时返回 500
- `messages` 非数组或为空时返回 400

**3. 调用百炼**

使用 `new OpenAI({ apiKey, baseURL })`，`openai.chat.completions.create({ model, messages, stream: true })`，获取流式 AsyncIterable。

**4. 转发 SSE**

`for await (const chunk of stream)` 遍历，取 `chunk.choices?.[0]?.delta?.content`，写入 `data: {"content":"..."}\n\n`，最后写入 `data: [DONE]\n\n` 并 `res.end()`。

**5. 异常处理**

捕获错误后返回 500 及 `error` 字段，前端按错误流程展示。

### 五、端到端数据流

| 阶段 | 数据/动作 | 代码位置 |
|------|-----------|----------|
| 1 | 用户点击发送 | ChatBot `handleSend` |
| 2 | Store 写入 user + 空 bot 消息 | `chatStore.addMessage` |
| 3 | 构造 `apiMessages` | `buildApiMessages` |
| 4 | `fetch('/api/chat', { body })` | `chat.ts` |
| 5 | Vite 转发到 BFF | vite.config proxy |
| 6 | BFF 校验并调用百炼 | server/index.js |
| 7 | 百炼返回流式 chunks | OpenAI SDK `for await` |
| 8 | BFF 写入 SSE 行 | `res.write("data: {...}\n\n")` |
| 9 | 前端 ReadableStream 读取 | `resp.body.getReader()` |
| 10 | 解析 SSE 调用 `onChunk` | `chat.ts` 行解析 |
| 11 | 追加到 bot 消息 | `appendToLastBotMessage` |
| 12 | 界面流式展示 | Pinia 响应式更新 |

### 六、安全

- API Key 仅存于 BFF 的 `.env`，不传给前端。
- 浏览器无法直接访问百炼 API。
